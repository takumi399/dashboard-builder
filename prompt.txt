## Phase 2：SQL 数据源

### 任务 1：后端 — 扩展 DataSource 模型

修改 backend/app/models/dashboard.py：
- DataSource 新增字段 `connection_config` (Text, nullable, 存储 JSON 连接信息)
- source_type 现在支持: "csv", "sql"

### 任务 2：后端 — 新增 SQL 连接配置 Schema

修改 backend/app/schemas/datasource.py：
- 新增 `SQLConnectionConfig` schema：
  - db_type: str (mysql / postgresql)
  - host: str
  - port: int
  - database: str
  - username: str
  - password: str
- 新增 `DataSourceSQLCreate` schema：
  - name: str
  - source_type: str = "sql"
  - connection_config: SQLConnectionConfig
- 新增 `SQLExecuteRequest` schema：
  - datasource_id: int
  - query: str

### 任务 3：后端 — SQL 执行端点

修改 backend/app/api/datasources.py：
- 新增 POST /api/datasources/sql/execute 端点
- 接收 SQLExecuteRequest
- 根据 datasource_id 获取连接配置
- 使用 SQLAlchemy create_engine 动态创建连接（同步即可，用 run_in_executor 或直接用同步 engine）
- 执行 SQL 查询，返回 {"columns": [...], "rows": [...], "row_count": N}
- SQL 注入防护：禁止 DROP/ALTER/TRUNCATE/DELETE/UPDATE/INSERT 关键字（只允许 SELECT）
- DDL 黑名单检查不通过返回 400

### 任务 4：后端 — 新增测试

在 backend/tests/test_datasources.py 添加：
- test_create_sql_datasource：创建 SQL 数据源（可以用 SQLite 文件数据库代替，避免依赖外部 MySQL）
- test_sql_execute：执行 SELECT 1 等简单查询
- test_sql_ddl_blocked：执行 DROP TABLE 应该被拦截返回 400

测试用的 SQL 数据源可以指向 SQLite 文件：
```python
connection_config = {
    "db_type": "sqlite",
    "database": ":memory:"  # 或临时文件
}
```
