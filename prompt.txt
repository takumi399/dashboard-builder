Fix the resize handle in C:\Users\Administrator\Desktop\claude\task\frontend\src\pages\DashboardEditorPage.tsx.

Problem: @dnd-kit's PointerSensor listens for pointer events, but the resize handle uses onMouseDown with stopPropagation. This doesn't stop pointer events from bubbling up to the parent div that has dnd-kit's drag listeners. So dragging the resize handle also moves the chart.

Fix: Change the resize handle's event from onMouseDown to onPointerDown with e.stopPropagation() and e.nativeEvent.stopImmediatePropagation(). This blocks the pointer event from reaching dnd-kit's sensor.

Steps:
1. Read the DraggableChart component in C:\Users\Administrator\Desktop\claude\task\frontend\src\pages\DashboardEditorPage.tsx
2. Find the resize handle div (around line 77)
3. Change `onMouseDown={handleResizeMouseDown}` to use onPointerDown instead
4. In handleResizeMouseDown, change the event type from React.MouseEvent to React.PointerEvent
5. Add `e.nativeEvent.stopImmediatePropagation()` after `e.stopPropagation()` to also block native pointer event propagation
6. Verify: cd C:\Users\Administrator\Desktop\claude\task\frontend && npx tsc --noEmit
7. Report result as JSON with what was changed
