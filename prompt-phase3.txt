## Phase 3：实时协作编辑

### 任务 1：后端 — WebSocket 端点

创建 backend/app/api/ws.py：
```
WebSocket 端点: /ws/dashboards/{dashboard_id}
```

功能：
- 连接时验证 JWT token（通过 query param `?token=xxx`）
- 维护一个内存字典：`{dashboard_id: set[WebSocket]}`
- 用户加入房间：广播 `{"type": "user_joined", "user_id": ..., "username": ...}`
- 用户离开：广播 `{"type": "user_left", "user_id": ...}`
- 转发操作：收到操作消息 → 广播给房间内其他用户
  - 操作类型：chart_added, chart_moved, chart_resized, chart_deleted, chart_updated
  - 消息格式：`{"type": "chart_moved", "chart_id": ..., "position_x": ..., "position_y": ...}`
- 使用 asyncio.Lock 保护房间字典的并发访问
- 新用户加入时，发送当前在线用户列表

更新 backend/app/main.py：
- 导入并注册 ws router
- 添加 WebSocket 端点

### 任务 3：前端 — useWebSocket Hook

创建 frontend/src/hooks/useWebSocket.ts：
```
参数：dashboardId, token
返回：{ isConnected, onlineUsers, sendOperation }
```

功能：
- 连接 ws://localhost:8000/ws/dashboards/{id}?token={token}
- 自动重连（断开后 3 秒重试，最多 5 次）
- 解析收到的消息，更新 onlineUsers 状态
- onmessage 回调分发给业务层
